**六、总结**

本次渗透测试我从外部 Web
资产入手，通过目录枚举发现备份文件与关键接口暴露风险，并进一步利用 SQL
注入漏洞获取数据库访问权限，成功提取系统用户凭据。在此基础上，我继续通过文件上传功能实现RCE，获得
Web 服务器低权限 Shell，并完成初始立足点建立。

我在对内网环境的侦察与路由分析，识别出域控主机与关键服务节点。利用服务器泄露的敏感信息文件，获取有效域用户凭据，然后通过
SMB 协议实现横向移动，成功登录域控主机并提升至 SYSTEM
权限。在域控环境中进一步执行凭据提取操作，获取域内高权限账户哈希信息，实现对
Active Directory 的完全控制能力。

在完全实现域控后，我创建高权限后门账户（backdoor），配合生成的Kerberos黄金票据，建立持久化访问路径，这样即使部分漏洞修复或入口被关闭，我仍可重新进入内网环境。整体攻击链覆盖从初始访问、权限提升、横向移动到域控完全接管的全过程，验证了目标系统在身份认证、访问控制及日志监测方面存在系统性安全风险。
